Stylesheet & CSS Frameworks

Bootstrap • TailwindCSS • Bulma

No Node.js

Bootstrap

Qué es: framework CSS/JS con grid, utilidades y componentes.

Ventajas:

  • maduro, docs y comunidad grandes
  • grid + utilidades listas
  • CDN, sin build

Desventajas:

  • look opinado/reconocible
  • sobreescrituras verbosas
  • plugins con Popper

Notas personales

Prototipos express y handoff con equipos no técnicos.

Node.js

Tailwind CSS

Qué es: utility-first CSS con purge/treeshake en build.

Ventajas:

  • veloz en HTML; diseño consistente
  • CSS final pequeño
  • config = design tokens

Desventajas:

  • requiere Node + PostCSS
  • HTML con muchas clases
  • cambio de mentalidad (utilities)

Notas personales

Stack favorito en React/Next por control y escalabilidad.

No Node.js

Bulma

Qué es: framework CSS puro basado en Flexbox.

Ventajas:

  • ligero; sin JS
  • estética limpia
  • CDN directo

Desventajas:

  • menos componentes JS listos
  • ecosistema menor
  • custom profundo vía Sass

Notas personales

Ideal cuando quiero simpleza y cero dependencias JS.

UI Kits & Component Libraries

shadcn/ui • Ark UI

Node.js

shadcn/ui

Qué es: componentes React copiables (Radix + Tailwind).

Ventajas:

  • accesibles y de alta calidad
  • código en tu repo (control)
  • excelente con Next/Tailwind

Desventajas:

  • Node + React obligatorios
  • acoplamiento a Tailwind
  • actualizaciones manuales

Notas personales

Para dashboards/portales con look actual y time-to-market corto.

Node.js

Ark UI

Qué es: componentes headless accesibles (React/Vue/Solid) sobre máquinas de estado.

Ventajas:

  • libertad total de estilos
  • accesibilidad sólida
  • multi-framework

Desventajas:

  • más trabajo de CSS
  • requiere runtime de framework
  • ecosistema más joven

Notas personales

Perfecto para construir un design system con identidad fuerte.

Animation & Visual Libraries

GSAP • AOS

No Node.js

GSAP

Qué es: motor JS de animación de alto rendimiento.

Ventajas:

  • control fino + plugins
  • performance probada
  • CDN o npm

Desventajas:

  • curva (timelines/ScrollTrigger)
  • plugins premium bajo licencia
  • más peso que CSS puro

Notas personales

Para microinteracciones premium y storytelling.

No Node.js

AOS (Animate On Scroll)

Qué es: animaciones on-scroll por data-attributes.

Ventajas:

  • setup mínimo (CDN)
  • ligero
  • sin build

Desventajas:

  • efectos limitados
  • poco control en escenas complejas
  • timelines largos difíciles

Notas personales

Para landings de marketing donde “algo de movimiento” basta.

Critical Reflection

Which tools fit my workflow & key trade-offs

Which tools fit your workflow?

Para productos y dashboards me quedo con Tailwind y shadcn/ui ya que son de maquetación muy rápida, diseño consistente por configuración y escalado limpio en equipos. Para dar “cara de estudio” a mis piezas y elevar la percepción, usaría GSAP en microinteracciones y storytelling (scroll, timelines, SVG), que aporta control fino y suavidad. Si necesitara entregar sin pipeline de build, Bootstrap o Bulma me permitirían salir con CDN y una base estable, ideal en prototipos docentes o sites corporativos sencillos. Para landings que solo requieren un toque de movimiento, AOS resuelve con casi cero complejidad. Y si necesito un punto de partida en horas, HTML5 UP me da un MVP estático que luego refactorizo a componentes.

Trade-offs observed

En velocidad vs flexibilidad, AOS y HTML5 UP maximizan el time-to-first-deliverable pero limitan el tema de personalización. GSAP y Tailwind exigen algo más de setup a cambio de control total. En dependencias vs simplicidad, shadcn y Ark incrementan productividad en stacks modernos (Node y framework), mientras Bulma/Bootstrap simplifican al prescindir de build, a costa de menos “a medida”. En Node.js vs standalone, Tailwind/shadcn/Ark brillan en productos escalables con pipeline; GSAP/AOS/Bootstrap/Bulma/HTML5 UP funcionan directo por CDN, perfectos para estáticos rápidos. En creatividad vs convención, GSAP/Tailwind favorecen identidad visual propia; Bootstrap prioriza consistencia y patrones reconocibles.

How would you use these tools in your portfolio?

Los case studies los construiría con Next, Tailwind y shadcn/ui para una UI coherente, accesible y mantenible (dark/light incluidos). Reservo GSAP para héroes, transiciones y microinteracciones que demuestren criterio de motion y storytelling. Para explicar conceptos o clases, Bulma/Bootstrap me permiten prototipar sin fricción y centrarme en el contenido. En campañas con plazos cortos, AOS aporta animación discreta y efectiva. Cuando el objetivo es validar rápido, parto de HTML5 UP, entrego el MVP y, si escala, migro a una arquitectura por componentes con Tailwind.
